配置项 含义
main.py 程序入口
datas 打进包的数据文件（如 lstm_artifacts）
hiddenimports 手动声明 PyInstaller 抓不到的 Python 模块
collect_all("chromadb") 整包收集 ChromaDB（否则 exe 启动报缺模块）
name="FaultDiagnosisAPI" 生成的 exe 名称
console=True 保留黑色命令行窗口（方便看日志）

常见补依赖位置： 若 exe 报 ModuleNotFoundError: No module named 'xxx'，在 spec 的 hiddenimports 列表里加上 "xxx"，然后重新打包。

3.4 目标电脑还需要什么

服务 用途 说明
Neo4j 知识图谱 需单独安装并启动，地址与 .env 一致
Ollama 大模型问答 需单独安装并拉取模型
MySQL 电压/电流历史数据 仅数据库相关功能需要

exe 不包含 以上服务，只是「后端接口程序」。

4、其他使用说明

操作 做法
启动后端 运行 FaultDiagnosisAPI.exe，保持黑窗口打开
停止后端 在黑窗口按 Ctrl + C，或直接关窗口
改端口 编辑 exe 同目录 .env 中的 PORT
改数据库 编辑 database.env 后重启 exe
更新程序 重新打包，用新文件夹整体替换旧文件夹（注意备份 data 和配置文件）

新增功能/路由时要改哪里？

若以后加了新的 Python 包或路由，打包可能失败，需同步更新：

情况 改哪个文件 怎么做
新增 FastAPI 路由文件 fault_diagnosis.spec 在 hiddenimports 加 "routers.新文件名"
新增 service 模块 fault_diagnosis.spec 在 hiddenimports 加 "services.新模块名"
新增需随程序发布的数据文件夹 fault_diagnosis.spec 在 datas 里追加 (源路径, 包内相对路径)
新增「可写」运行时目录 app_paths.py + 业务代码 参照 get_lstm_artifacts_dir() 写法
新增环境变量配置 env.packaged.example 加示例项，并更新部署说明
