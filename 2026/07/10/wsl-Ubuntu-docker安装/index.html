docker打包

Package files：要打包的整个程序文件夹

environment.yml：由conda输出的环境配置文件，里面会列出所有用到的依赖项

Dockerfile： 一步步把依赖库和启动程序构建好的脚本，别人run我们的镜像image时，就会根据这个脚本启动

1.将Package files放到要打包的项目中

2.打包成镜像docker build -t [镜像名] .

3.保存tar包：docker save -o 想要的包名.tar [镜像名]

部署到客户机

1.用户电脑上需安装好wsl、Ubuntu、docker

2.解压tar包：docker load -i xxx.tar

3.启动镜像：docker run -p 0.0.0.0:8001:8001 –name <容器名称> kg_gou:v1（打包的镜像名）

4.停止使用镜像：docker stop <容器名称>
