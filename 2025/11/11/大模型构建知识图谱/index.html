1.ollama下载安装

ollama安装

进入ollama官网，点击下载后得到OllamaSetup.exe。

Ollama下载 - 官方正版 - 极客应用网站下载比较快

直接点击exe文件默认下载到C盘，模型也都存储在C盘

将这个exe文件放到想要安装ollama的位置（D:\ollamaAI）。然后在这个文件夹的位置打开cmd，输入以下命令，将ollama下载到非c盘的位置。

下载完成后可以进入ollama文件路径下的cmd（D:\ollamaAI\ollama)，检查ollama服务。

模型下载

修改下载模型的路径。

自定义Ollama安装路径

回到ollama官网中，点击model，找到想下载的模型，点击进入模型的详情页。

然后在cmd中运行ollama run deepseek-r1:7b即可下载模型。(最新版的deepseek能进行语言解析，但是没办法给出三元组)

下载成功后进入ollama文件路径下的cmd：ollama run deepseek-r1:7b 测试模型能否运行

启动Ollama服务：ollama serve

2.项目环境

  • Python 3.8
  • Neo4j 数据库（已安装并运行）
  • Ollama 或其他 LLM 服务（用于文本解析）

后端安装包

下载项目ai-knowledge-gra后创建虚拟环境conda create -n AIKG python=3.11后通过 conda activate AIKG进入该项目所在的环境。

使用pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple安装所有需要的库

requirement.txt中库不全，继续安装pip install flask flask-cors neo4j toml

3.项目运行

如果第一次运行，先运行 python generate_and_import_neo4j.py 导入示例数据

目前 api_server.py 的 /api/upload 接口只提取三元组并返回，未自动导入 Neo4j。要实现“存入 Neo4j”，需要：

  1. 修改 api_server.py，在提取三元组后调用 tupu.py 的 import_to_neo4j 函数

  2. 或者在前端上传后添加一个“保存到数据库”按钮，调用新的 API 接口

注意修改generate_and_import_neo4j.py，api_server.py文件中neo4j的账户密码

后端在终端输入python api_server.py

前端在终端输入npm run dev

4、常见问题

Q: 后端无法连接 Neo4j？ A: 检查 Neo4j 是否运行，确认密码配置正确 B.查看项目环境是否是AIKG

Q: 前端页面空白？ A: 检查后端是否在 http://localhost:5000 运行

Q: 文件上传失败？ A: 确保 Ollama 或其他 LLM 服务正在运行（用于文本解析）
