Props（父 → 子组件通信）

基本概念

Props 是父组件向子组件传递数据的单向数据流机制，具有以下特点：

  • 单向数据流：只能从父组件流向子组件
  • 显式声明：子组件需要明确声明接收的 props
  • 类型检查：可以指定 props 的类型和验证规则

1. 父组件 (EditView.vue) 传递 Props

  • :student-data：动态绑定父组件的 studentData 到子组件
  • :is-editable：动态绑定编辑状态布尔值

2. 子组件 (StuCard.vue) 声明 Props

Props 的高级特性

  1. 类型验证：

    • Vue 会检查传入的 prop 值是否符合声明的类型
    • 支持多种类型：type: [String, Number]

  2. 默认值：

    • 当父组件没有传递 prop 时使用默认值
    • 对象/数组必须用工厂函数返回：default: () => []

Emits（子 → 父组件通信）

基本概念

Emits 是子组件向父组件通信的方式：

  • 事件触发：子组件通过 emit 触发事件
  • 事件监听：父组件通过 v-on 或 @ 监听事件
  • 数据传递：可以携带任意数据

Ref 方式 vs Props/Emits

特性 Ref 方式 Props/Emits 方式
通信方向 父→子 父↔子
耦合度 高（需要知道子组件具体实现） 低（通过接口通信）
适用场景 需要主动控制子组件或获取内部状态 标准的组件通信
可维护性 较低（实现细节耦合） 较高（接口清晰）
实时性 可随时获取最新状态 需要子组件主动通知
